17 /**
18 * @brief Base class representing an interactive game entity in the game world.
19 *
20 * @details The `GameObject` provides a game-logic-level abstraction over a `SceneNode`.
21 * It serves as the primary building block for game-specific entities like players or enemies,
22 * designed to be subclassed to add further behavior. Each `GameObject` has a unique
23 * `Guid` for easy identification and provides convenient methods for manipulating the
24 * transform of its underlying `SceneNode`. It also acts as the target for `Command` execution.
25 *
26 * Example usage:
27 * ```cpp
28 * class Player : public helios::engine::game::GameObject {
29 * public:
30 * using GameObject::GameObject;
31 *
32 * void jump() {
33 * setTranslation({0.0f, 5.0f, 0.0f});
34 * }
35 * };
36 *
37 * auto sceneNode = std::make_shared<helios::scene::SceneNode>();
38 * auto player = std::make_unique<Player>(sceneNode.get());
39 * player->setTranslation({1.0f, 0.0f, 0.0f});
40 * ```
41 *
42 * @warning The GameObject does NOT own the SceneNode. The caller must ensure the SceneNode
43 * outlives the GameObject. If the SceneNode is destroyed while the GameObject is
44 * still alive, all transform operations will result in undefined behavior.
45 *
46 * @todo Implement observer pattern or weak_ptr to detect SceneNode destruction and
47 * invalidate the transform pointer to prevent dangling pointer issues.
48 */
49 class GameObject {
