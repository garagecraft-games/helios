17 /**
18 * @brief A lightweight class for transferring the state of a Gamepad.
19 *
20 * This class models a reusable object representing the input state of a gamepad.
21 * The input state of a gamepad consists of the state of the left and right trigger,
22 * ranging from [0, 1], where 1 means fully pressed and 0 means not pressed at all.
23 *
24 * Similarly, a GamepadState provides information about the left and the right
25 * gamepad axes. Both the x- and the y-axis range from [-1, 1]. For the x-axis,
26 * the following holds:
27 * - 0 => not moved at all
28 * - -1 => moved all the way to the left
29 * - 1 => moved all the way to the right
30 *
31 * For the y-axis the positive direction corresponds to "up" and the negative
32 * direction corresponds to "down".
33 *
34 * Updating the values is done by calling `updateAxes` on this object.
35 *
36 * For convenient access of the axes in 2D coordinates, the class provides accessors
37 * to the sticks' axes in `vec2f` form.
38 *
39 * Button states are generally represented with boolean values, i.e. true for pressed,
40 * otherwise false.
41 *
42 * @note Implementations must account for joystick drift: achieving exactly 0
43 * for x/y when no human input occurred is rare. Consider applying a dead zone
44 * when processing the GamepadState. Axis values are individually clipped to [-1.0, 1.0],
45 * but the magnitude of the resulting (x, y) vector may exceed 1.0. Applications should
46 * normalize input vectors where appropriate and possibly offer calibration.
47 *
48 * This implementation is inspired by the GLFW gamepad input model.
49 * @see https://www.glfw.org/docs/latest/input_guide.html#joystick
50 */

222 /**
223 * @brief Updates the axes and button states of this GamepadState object.
224 *
225 * This method updates raw axis and trigger values and stores the boolean
226 * state of all standard gamepad buttons. Float parameters are asserted to
227 * be in their expected ranges and then clamped.
228 *
229 * For the sticks' range [-1, 1] the following holds:
230 * -1 means moved all the way left/down, 0 means not moved at all, 1 means moved all the way right/up.
231 *
232 * For the triggers' range of [0, 1] the following holds:
233 * 0 means not pressed, 1 means fully pressed.
234 *
235 * @param axisLeftX The x-axis of the left stick, expected in [-1, 1].
236 * @param axisLeftY The y-axis of the left stick, expected in [-1, 1].
237 * @param axisRightX The x-axis of the right stick, expected in [-1, 1].
238 * @param axisRightY The y-axis of the right stick, expected in [-1, 1].
239 * @param triggerLeft The left trigger value, expected in [0, 1].
240 * @param triggerRight The right trigger value, expected in [0, 1].
241 * @param buttonA True if the A button is pressed.
242 * @param buttonB True if the B button is pressed.
243 * @param buttonX True if the X button is pressed.
244 * @param buttonY True if the Y button is pressed.
245 * @param buttonStart True if the Start button is pressed.
246 * @param buttonBack True if the Back button is pressed.
