23 /**
24 * @brief InputManager providing access to various input states.
25 *
26 * The `InputManager` acts as an intermediary between the application and
27 * the underlying systems that provide input. Raw events are processed by
28 * InputAdapters owned by this `InputManager`.
29 *
30 * An `InputManager` allows querying `GamepadState` objects by calling
31 * `gamepadState()`. The method accepts an id that identifies the
32 * gamepad whose input state should be returned.
