14 /**
15 * @brief Abstract base class for encapsulating game actions as executable commands.
16 *
17 * @details A `Command` encapsulates a single game action (e.g., movement, attack)
18 * to be performed on a `GameObject`.
19 *
20 * Concrete commands are typically created by an `InputHandler` and queued in a
21 * `CommandBuffer`. The buffer then executes them against the target `GameObject`
22 * during a later processing step.
23 *
24 * Example implementation:
25 * ```cpp
26 * class MoveCommand : public helios::engine::game::Command {
27 * helios::math::vec3f direction_;
28 * float speed_;
29 * public:
30 * MoveCommand(helios::math::vec3f dir, float speed)
31 * : direction_(dir), speed_(speed) {}
32 *
33 * void execute(helios::engine::game::GameObject& obj) override {
34 * obj.translate(direction_ * speed_);
35 * }
36 * };
37 * ```
38 *
39 * @note Implementations should ensure execute() is exception-safe or noexcept,
40 * as throwing exceptions during CommandBuffer::flush() will abort execution
41 * of remaining commands.
42 */
