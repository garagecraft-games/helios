36 /**
37 * @brief Represents a SceneNode within a SceneGraph.
38 *
39 * A SceneNode holds its local transformation and a (shared) pointer to
40 * a Renderable object (the representative of a visible element).
41 * Furthermore, a SceneNode can have multiple child nodes uniquely owned by
42 * **this** SceneNode.
43 *
44 * A SceneNode holds a global unique identifier and tracks a dirty state (`needsUpdate_`)
45 * related to any transformations that occurred on the SceneNode.
46 * Ownership is managed by a collection of unique pointers of the child nodes.
47 * SceneNodes are designed to be not copyable and not assignable to prevent duplication
48 * and to enforce the hierarchical tree structure.
49 *
50 * The `worldTransform_` is derived from a SceneNode's `localTransform_` and the SceneNode's
51 * parent's `worldTransform`s. A call to `worldTransform()` will update the particular subtree
52 * of the SceneNode bottom-up to make sure the actual values are available with the SceneNode's
53 * `worldTransform`. A Scene might also propagate changes top-bottom - see `Scene:.udpateNodes()`.
54 *
55 */
56 class SceneNode {
57

59 /**

64
65 protected:
66 /**
67 * @brief Boolean flag to represent the dirty-state of this node.
68 * Should be set to false if the current `worldTransform_`
69 * for this node was computed, otherwise true, e.g. if the node's
70 * local transform has been changed.
71 */
72 bool needsUpdate_ = true;
