22 /**
23 * @brief Represents a single RenderPass under a set of specific uniform values.
24 *
25 * A `RenderPass` holds a `RenderQueue` that contains all `RenderCommands` that need
26 * to be processed in the subsequent rendering. The associated uniform values specified
27 * with this `RenderPass` remain constant during processing of the `RenderCommand`s.
28 *
29 * `RenderPass`es are typically processed sequentially by an API-specific implementation
30 * of the pure virtual `RenderingDevice` class.
31 *
32 * @note A `RenderPass` should be configurable with other properties relevant to the underlying
33 * rendering backend, e.g. depth testing, draw mode etc.
34 */
