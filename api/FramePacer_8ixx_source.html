16 /**
17 * @class FramePacer
18 * @brief Controls and maintains a target frame rate through precise timing and pacing.
19 *
20 * The FramePacer class utilizes a `Stopwatch` to measure frame execution time and
21 * introduces necessary sleep delays to maintain a consistent target frame rate.
22 * It returns detailed timing statistics via the `FrameStats` structure upon synchronization.
23 *
24 * @details The pacing mechanism helps in achieving smoother frame delivery by minimizing
25 * jitter, although strict adherence depends on the OS scheduler's precision.
26 *
27 * @note If `targetFps` is set to 0.0f, no frame pacing is performed, and frames
28 * run as fast as the hardware and OS allow (unlocked framerate).
29 *
30 * @par Usage Example:
31 * ```cpp
32 * auto stopwatch = std::make_unique<helios::util::time::Stopwatch>();
33 * FramePacer pacer(std::move(stopwatch));
34 * pacer.setTargetFps(60.0f);
35 *
36 * while (running) {
37 * pacer.beginFrame();
38 * // ... game logic and rendering ...
39 * FrameStats stats = pacer.sync();
40 * }
41 * ```
42 */
43 class FramePacer {

95 /**
96 * @brief Synchronizes frame timing and returns frame statistics.
97 *
98 * Measures the elapsed work time since `beginFrame()`. If a target FPS is set
99 * and the work time is less than the target frame duration, this method
100 * sleeps the current thread to meet the target timing.
101 *
102 * @return A `FrameStats` structure containing the total frame time (including wait),
103 * the actual work time (CPU processing), and the wait time (idle).
104 *
105 * @note If `targetFps` is 0.0f or the frame took longer than the target duration,
106 * no sleeping occurs, and `waitTime` in the returned stats will be 0.0f.
107 *
108 * @todo Implement hybrid spinning for the last millisecond of the wait time
109 * to improve timing precision and mitigate OS scheduler wake-up latency.
110 */
111 [[nodiscard]] FrameStats sync();
