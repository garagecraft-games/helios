24 /**
25 * @brief Queue for deferred execution of game commands.
26 *
27 * @details The `CommandBuffer` provides a queue for game commands, decoupling their
28 * creation from their execution. Commands are buffered along with their target
29 * `GameObject` identifiers (`Guid`) and are executed in a single batch when `flush()`
30 * is called against a `GameWorld`.
31 *
32 * Typical usage pattern:
33 * ```cpp
34 * helios::engine::game::CommandBuffer cmdBuffer;
35 * helios::engine::game::GameWorld world;
36 *
37 * // Queue commands during input processing
38 * cmdBuffer.add(player.guid(), std::make_unique<MoveCommand>(direction, speed));
39 * cmdBuffer.add(enemy.guid(), std::make_unique<AttackCommand>(target));
40 *
41 * // Execute all commands at end of frame
42 * cmdBuffer.flush(world); // Executes and clears buffer
43 * ```
44 *
45 * @note Commands targeting non-existent GameObjects are silently skipped during flush().
46 * This behavior will be logged once the logging system is available.
47 */
48 class CommandBuffer {
