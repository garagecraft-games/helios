22 /**
23 * @brief Central registry and owner of all active GameObjects in the game world.
24 *
25 * @details The `GameWorld` manages the lifetime and lookup of all `GameObject` instances.
26 * It provides fast, `Guid`-based lookup and handles the lifecycle of game
27 * entities through a clear ownership model. The world serves as the central registry for all
28 * objects and is used by the `CommandBuffer` to execute commands against them.
29 *
30 * Typical usage:
31 * ```cpp
32 * helios::engine::game::GameWorld world;
33 *
34 * // Add a new entity
35 * auto player = std::make_unique<Player>(sceneNode);
36 * auto* playerPtr = world.addGameObject(std::move(player));
37 *
38 * // Lookup by Guid
39 * if (auto* obj = world.find(playerPtr->guid())) {
40 * obj->setTranslation({1.0f, 0.0f, 0.0f});
41 * }
42 *
43 * // Remove entity (returns ownership)
44 * auto removed = world.removeGameObject(*playerPtr);
45 * ```
46 *
47 * @note GameWorld owns all GameObjects added to it. When a GameObject is removed,
48 * ownership is transferred back to the caller via std::unique_ptr.
49 */
