Detailed Description

Represents a single RenderPass under a set of specific uniform values.

A `RenderPass` holds a `RenderQueue` that contains all `RenderCommands` that need to be processed in the subsequent rendering. The associated uniform values specified with this `RenderPass` remain constant during processing of the `RenderCommand`s.

`RenderPass`es are typically processed sequentially by an API-specific implementation of the pure virtual `RenderingDevice` class.

Note
A `RenderPass` should be configurable with other properties relevant to the underlying rendering backend, e.g. depth testing, draw mode etc.
