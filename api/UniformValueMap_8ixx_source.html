31 /**
32 * @brief Manages a collection of values for mapping them to their UniformSemantics
33 * identifier.
34 *
35 * This class allows for storing different types of uniform values (e.g. mat4f, vec3f, ...)
36 * under a single semantic identifier, for convenient assignment to uniform locations
37 * of an underlying shader in the rendering process.
38 *
39 * @note this class intentionally avoids templates as we assume that instances of the
40 * value maps are used in the rendering hot path.
41 *
42 * @todo UniformMap must allow only one index for all data structures,
43 * i.e. if map_[semantics] contains a mat4f, other types should not be allowed
44 * for the same semantics.
45 *
46 * @todo UniformValueMaps should be scoped, e.g. per frame (world matrix,
47 * projection, view...), per material (colors, emissive...), per object (world matrix...),
48 * which allows for better handling of assigning uniforms: per-frame values change once per frame,
49 * while per-object values change per object.
50 */
